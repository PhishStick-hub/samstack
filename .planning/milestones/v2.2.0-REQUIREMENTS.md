# Requirements: samstack — v2.2.0 Per-Function Warm Containers

**Defined:** 2026-04-25
**Archived:** 2026-04-25
**Core Value:** No leftover Docker containers or networks after a crashed pytest session.

## v2.2.0 Requirements (All Complete)

### Configuration

- [x] **CONF-01**: User can specify a list of functions to pre-warm via `SamStackSettings` / `pyproject.toml`
- [x] **CONF-02**: User can override the pre-warm function list via a `warm_functions` fixture
- [x] **CONF-03**: Pre-warming is a no-op when the function list is empty (backward compatible)

### start-lambda Pre-Warm

- [x] **LAMBDA-01**: `start-lambda` functions can be pre-warmed via `lambda_client.invoke()` before any test executes
- [x] **LAMBDA-02**: Pre-warming runs sequentially to avoid race conditions
- [x] **LAMBDA-03**: Pre-warm failures raise `SamStartupError` by default (hard fail)

### start-api Pre-Warm

- [x] **API-01**: `start-api` functions can be pre-warmed via HTTP synthetic requests before any test executes
- [x] **API-02**: Users provide explicit API route URLs for HTTP pre-warming (no template parser)

### Testing & Documentation

- [x] **TEST-01**: Integration tests verify that pre-warmed functions are actually warm
- [x] **TEST-02**: Crash tests verify warm sub-containers are cleaned up on SIGKILL
- [x] **TEST-03**: Migration guide documents the `EAGER`→`LAZY` default behavior change
- [x] **TEST-04**: Documentation covers configuration, fixture overrides, and limitations

## Deferred Requirements (v3+)

- **AUTO-01**: Auto-discovery of functions from SAM template for pre-warming
- **HEAL-01**: Warm container health check / retry with configurable backoff
- **STAT-01**: Warm container status introspection fixture for diagnostics
- **RESET-01**: Function-scoped container reset / cold-start forcing
- **TMPL-01**: Template-aware API route discovery for `start-api` pre-warming

## Out of Scope

| Feature | Reason |
|---------|--------|
| Auto-discovery of API routes from SAM template | Deferred to future — CloudFormation intrinsics make robust parsing complex |
| Per-function container reset / cold-start forcing | Too fragile and niche; SAM CLI has no native support |
| Warm container introspection/status fixture | Diagnostic feature, not core to per-function warm control |
| `--debug-port` compatibility with warm containers | SAM CLI limitation (issue #7308); explicitly documented exclusion |
| Multiple SAM template instances per session | Architectural overkill |
