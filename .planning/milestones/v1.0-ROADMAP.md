## Overview

A targeted fix ensuring all samstack-managed Docker containers and networks are crash-safe via testcontainers Ryuk reaper integration. Phase 1 wired the network with Ryuk labels and TCP socket registration. Phase 2 verified all three main container fixtures (LocalStack, SAM API, SAM Lambda) carry the `org.testcontainers.session-id` label. Phase 3 empirically verified that SAM-spawned Lambda runtime sub-containers (created via Docker-in-Docker, without testcontainers labels) are cleaned up via network cascade on both normal teardown and crash paths.

## Phases

### Phase 1: Ryuk Network Wiring

**Goal**: The Docker bridge network created by samstack is crash-safe — Ryuk cleans it up on process death, normal teardown still works, and CI environments without Ryuk are unaffected
**Depends on**: Nothing (first phase)
**Plans**: 2 plans

Plans:
- [x] 01-01 — Wire Ryuk label + socket registration into `docker_network`; unit tests TEST-01, TEST-02
- [x] 01-02 — Automated SIGKILL crash test (TEST-03)

**Details:**
- Labeled `docker_network` with `LABEL_SESSION_ID: SESSION_ID` at creation
- Registered network with Ryuk via `network=name=<name>\r\n` TCP socket filter
- Gated Ryuk code behind `testcontainers_config.ryuk_disabled`
- Graceful degradation with `warnings.warn(stacklevel=2)` on socket failures
- Crash test: subprocess pytest session, SIGKILL, Docker API poll for network removal (TEST-03)

### Phase 2: Container-Level Ryuk Verification

**Goal**: Empirically verify that all three samstack container fixtures (LocalStack, SAM API, SAM Lambda) carry the Ryuk session label after `.start()`
**Depends on**: Phase 1
**Plans**: 1 plan

Plans:
- [x] 02-01 — Verify LocalStack and SAM container Ryuk labels via Docker SDK label queries

**Details:**
- Label inspection only — no crash test for containers
- Docker SDK `containers.list(filters={"label": "org.testcontainers.session-id"})` for SAM containers
- LocalStack checked via `get_wrapped_container().labels` (direct handle)
- CI gating: same `pytestmark` skip guard pattern from TEST-03

### Phase 3: Sub-Container Cascade Verification

**Goal**: Empirically verify SAM Lambda runtime sub-containers (spawned by SAM CLI via DinD, without testcontainers labels) are cleaned up when the Docker bridge network is removed on both normal teardown and crash paths
**Depends on**: Phase 2
**Plans**: 2 plans

Plans:
- [x] 03-01 — Extend crash test (TEST-03) with full SAM session + hard sub-container assertion (30s timeout, `sam_` name filter)
- [x] 03-02 — Normal teardown sub-container verification (15s timeout, runs on all platforms)

**Details:**
- Crash test upgraded from `cascade_note` print to `assert sub_containers_gone`
- `_write_subprocess_session` extended to include full SAM build + Lambda invocation via hello_world fixture
- New `_poll_containers_gone()` helper added (mirrors `_poll_until_gone` pattern)
- Normal teardown test: subprocess launches SAM, invokes Lambda, exits cleanly, parent asserts zero `sam_` containers
- Crash path: macOS skip preserved (Docker Desktop TCP proxy limitation)
- Teardown path: runs on all platforms (tests `_teardown_network`, not Ryuk)

## Milestone Summary

**Key Decisions:**
- Ryuk integration over pre-session cleanup — Ryuk fires immediately on process death
- All samstack containers targeted via network-level cleanup — simplest complete solution
