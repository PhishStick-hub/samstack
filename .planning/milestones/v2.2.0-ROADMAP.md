# Milestone v2.2.0: Per-Function Warm Containers

**Status:** ✅ SHIPPED 2026-04-25
**Phases:** 4-7
**Total Plans:** 6

## Overview

Give testers granular control over which Lambda functions get warm containers and when they start. Per-function warm container configuration via `SamStackSettings.warm_functions` and overridable pytest fixtures. Selective pre-warming for both `start-lambda` (boto3 invoke) and `start-api` (HTTP GET) paths. Integration tests with init-marker UUID pattern proving container reuse. Warm sub-container crash test verifying Ryuk cascade cleanup.

## Milestone Summary

**Key Decisions:**
- warm_functions: bare `list[str]`, default `[]`, overridable session fixture
- start-lambda: LAZY when warm_functions non-empty, EAGER when empty (backward compatible)
- start-api: always LAZY (unchanged), opt-in pre-warm via warm_api_routes fixture
- Pre-warm: sequential, single attempt, hard-fail via SamStartupError
- Init-marker UUID pattern for deterministic container reuse verification
- All docs live in README.md — no separate MIGRATION.md

**Issues Resolved:**
- Dynamic warm_containers switching (LAZY vs EAGER) in sam_lambda_endpoint
- HTTP pre-warm intersection filter: only functions in both warm_functions AND warm_api_routes
- Duplicate "Upgrading from v1.0" section removed post-UAT
