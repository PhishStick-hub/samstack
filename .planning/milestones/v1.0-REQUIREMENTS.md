# Requirements: v1.0 — Ryuk Network Wiring

**Defined:** 2026-04-23
**Shipped:** 2026-04-24
**Core Value:** No leftover Docker containers or networks after a crashed pytest session.

## v1 Requirements (All Complete)

### Ryuk Wiring

- [x] **RYUK-01**: Docker network created by `docker_network` fixture is labeled with `org.testcontainers.session-id` so Ryuk can identify it
  - Outcome: ✅ Implemented in `docker_network` fixture via `LABEL_SESSION_ID: SESSION_ID` label at creation time
- [x] **RYUK-02**: Docker network is explicitly registered with Ryuk reaper via `network=name=<name>\r\n` filter on the TCP socket
  - Outcome: ✅ Implemented with None-guarded `Reaper._socket.send()` in try/except block
- [x] **RYUK-03**: Ryuk registration is gated behind `testcontainers_config.ryuk_disabled` — no crash or test failure when Ryuk is disabled (CI environments)
  - Outcome: ✅ Verified by unit test `test_ryuk_disabled_skips_registration`
- [x] **RYUK-04**: Ryuk wiring degrades gracefully — socket failures emit `warnings.warn`, not exceptions
  - Outcome: ✅ Verified by unit test `test_socket_failure_warns_not_raises`
- [x] **RYUK-05**: Existing `_teardown_network` teardown path is preserved unchanged for normal-exit cleanup
  - Outcome: ✅ Original teardown block untouched; Ryuk registration added before yield only

### Verification

- [x] **TEST-01**: Unit test verifies `docker_network` creates a network with the correct session-id label
  - Outcome: ✅ `test_network_has_session_id_label` verifies label presence
- [x] **TEST-02**: Unit test verifies Ryuk registration is skipped when `ryuk_disabled=True`
  - Outcome: ✅ `test_ryuk_disabled_skips_registration` verifies bypass
- [x] **TEST-03**: Manual/integration crash test (SIGKILL pytest mid-run) confirms network and containers are removed; documents sub-container cascade behavior
  - Outcome: ✅ `test_network_removed_after_sigkill` hard-asserts network removal within 5s; cascade behavior documented in test output

All 8 v1 requirements shipped as planned. No requirements were adjusted or dropped during the milestone.
