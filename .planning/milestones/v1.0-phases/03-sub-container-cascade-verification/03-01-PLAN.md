**New assertion block** (replaces lines 136-150):

```python
# Hard-assert sub-container cascade per D-03/D-06:
# After Ryuk removes the network, SAM Lambda runtime sub-containers
# (prefixed "sam_") must be cleaned up. Docker daemon processes
# the network disconnect asynchronously, so poll with a generous
# timeout to allow containers to transition from running → exited → removed.
def _poll_containers_gone(
    client: docker.DockerClient,
    name_prefix: str,
    timeout: float = 30.0,
    interval: float = 0.5,
) -> bool:
    """Return True if zero containers matching name_prefix remain."""
    deadline = time.monotonic() + timeout
    while time.monotonic() < deadline:
        remaining = client.containers.list(
            all=True, filters={"name": name_prefix}
        )
        if not remaining:
            return True
        time.sleep(interval)
    return False

sub_containers_gone = _poll_containers_gone(
    docker_client, "sam_", timeout=30.0, interval=0.5
)
assert sub_containers_gone, (
    "SAM Lambda runtime sub-containers still exist 30 s after SIGKILL + "
    "network removal. Ryuk cascade did not clean up sub-containers. "
    f"Remaining: {[c.name for c in docker_client.containers.list(all=True, filters={'name': 'sam_'})]}"
)
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Docker daemon socket | Test code calls `docker.from_env()` which connects to the Docker socket at `/var/run/docker.sock`. Container listing and network queries cross this boundary. |
| Subprocess boundary | `subprocess.Popen` launches a pytest session in a child process. The parent sends SIGKILL via `os.kill()` and reads Docker state independently — no subprocess stdout parsing. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | `_write_subprocess_session` — generated conftest and test files written to `tmp_path` | accept | Files are written by the test itself to a temp directory. No external input crosses this boundary. Accept — test is self-contained. |
| T-03-02 | Information Disclosure | Docker socket access via `docker.from_env()` in test code | accept | Docker socket access is already required for the samstack plugin to function. The test queries container metadata (names, status) which is not sensitive. No credentials or secrets are read or stored. Accept. |
| T-03-03 | Denial of Service | Subprocess SIGKILL via `os.kill(proc.pid, signal.SIGKILL)` | accept | The subprocess is launched by the test itself in an isolated `tmp_path` directory. SIGKILL only affects the child process. No risk of killing unrelated processes. Accept. |
| T-03-04 | Elevation of Privilege | Subprocess inherits parent's Docker socket access | accept | The child pytest session uses the same Docker socket as the parent — this is by design (Docker-in-Docker path). No privilege boundary is crossed. The subprocess has the same access level as the parent. Accept. |
</threat_model>
