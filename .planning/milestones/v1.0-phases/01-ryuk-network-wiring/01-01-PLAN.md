<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| test process → Ryuk TCP socket | samstack sends a filter string over a local loopback socket owned by the Ryuk container |
| Docker SDK → Docker daemon socket | `docker_sdk.from_env()` uses `/var/run/docker.sock` |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | `network=name=<name>\r\n` filter string | accept | Network name is UUID-derived (`samstack-{uuid4().hex[:8]}`); no user-controlled input; low risk |
| T-01-02 | Denial of Service | Ryuk socket connection failure | mitigate | try/except around `Reaper._socket.send` with `warnings.warn` — session continues on failure (D-04) |
| T-01-03 | Elevation of Privilege | Docker socket access (`/var/run/docker.sock`) | accept | This is test-infrastructure code running in developer environments; socket access is a design requirement, not a flaw |
| T-01-04 | Information Disclosure | Warning message includes exception text | accept | Warning contains only `str(exc)` from Docker/socket errors; no secrets or PII |
</threat_model>
