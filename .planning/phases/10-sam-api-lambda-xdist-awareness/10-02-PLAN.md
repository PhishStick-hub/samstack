<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| gw0 → shared state file | gw0 writes `sam_lambda_endpoint` URL and `error` messages to JSON state file on temp filesystem |
| gw1+ ← shared state file | gw1+ reads `sam_lambda_endpoint` URL from JSON state file; trusts content written by gw0 |
| gw1+ ↔ SAM container | gw1+ sends boto3 Lambda invoke calls to gw0's SAM start-lambda container via shared endpoint URL |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-05 | Tampering | Shared state file (`sam_lambda_endpoint` key) | accept | Local-only temp directory; only gw0 writes to state; filelock prevents concurrent writes. No untrusted access to test host filesystem. |
| T-10-06 | Denial of Service | gw0 container failure → gw1+ hang | mitigate | gw0 writes `"error"` key on failure; gw1+ `wait_for_state_key()` detects error key and calls `pytest.fail()` within 120s timeout — same pattern as Phase 9 and Plan 01 |
| T-10-07 | Information Disclosure | Endpoint URL in shared state file | accept | URL is `http://127.0.0.1:{port}` — localhost only, no secrets, no PII. Lambda invoke payloads travel over loopback. |
| T-10-08 | Spoofing | Malicious worker writes fake endpoint | accept | Only `gw0` writes to state file by design; gw1+ path never calls `write_state_file`. In local test environment, no untrusted workers exist. |
</threat_model>
