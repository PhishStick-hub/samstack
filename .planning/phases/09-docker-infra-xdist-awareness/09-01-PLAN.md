<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Environment → `_xdist.py` | `PYTEST_XDIST_WORKER` env var determines worker role — no authentication boundary, testing tool context |
| State file → gw1+ | Shared JSON state file read by gw1+ — contains only internal Docker URLs, no secrets |
| Docker socket → localstack_container (gw0) | Docker API calls from gw0 only — gw1+ has no Docker access |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-01 | Spoofing | `localstack_container` gw0 role detection | accept | `PYTEST_XDIST_WORKER` env var is set by pytest-xdist, not user-controllable in normal test execution |
| T-09-02 | Tampering | Shared state file `localstack_endpoint` key | accept | State file in temp dir with UUID isolation; Docker URLs are internal-only, no secrets |
| T-09-03 | Information Disclosure | `localstack_endpoint` in shared state | accept | Contains only `http://127.0.0.1:PORT` — local loopback address, no sensitive data |
| T-09-04 | Denial of Service | gw1+ polling `wait_for_state_key` | mitigate | 120s timeout with 0.5s poll interval prevents infinite hangs; `error` key triggers `pytest.skip()` for fail-fast |
| T-09-05 | Elevation | `_LocalStackContainerProxy` on gw1+ | accept | Proxy is a pure data wrapper with no system calls; no privilege escalation path |
| T-09-06 | Denial of Service | gw0 Docker startup failure | mitigate | gw0 writes `error` key to state on any exception during startup; gw1+ detects via `wait_for_state_key` and skips within timeout |
</threat_model>
