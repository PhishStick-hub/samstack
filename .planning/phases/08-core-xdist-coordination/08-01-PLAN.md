Create `src/samstack/_xdist.py` with the following functions (import-safe — no external deps imports in the main body, lazy imports where needed):

**Worker detection (stdlib only, import-safe):**
- `get_worker_id() -> str`: Reads `os.environ.get("PYTEST_XDIST_WORKER", "master")`. Returns `"master"` (no xdist), `"gw0"`, `"gw1"`, etc. This is the ONLY import-safe way to detect xdist context — no `import xdist` required.
- `is_xdist_worker(worker_id: str | None = None) -> bool`: Returns `True` if worker_id starts with `"gw"` AND is not `"master"`. When `worker_id` is None, calls `get_worker_id()`. This means `"gw0"` IS a worker (returns True) — gw0 runs tests like any other worker.
- `is_controller(worker_id: str | None = None) -> bool`: Returns `True` when worker_id is `"master"` (no xdist) OR `"gw0"` (xdist, first worker is infrastructure owner). Returns `False` for gw1+. This function answers "should THIS worker create Docker infrastructure?"

**Session isolation:**
- `get_session_uuid() -> str`: Generates a UUID4 hex string (8 chars), caches it in a module-level `_session_uuid` variable. This ensures all calls within the same process return the same UUID, but different pytest sessions get different UUIDs. Used for state file directory and lock file paths.
- `get_state_dir() -> Path`: Returns `Path(tempfile.gettempdir()) / f"samstack-{get_session_uuid()}"`. Creates the directory with `parents=True, exist_ok=True`. Uses `tempfile.gettempdir()` which respects `TMPDIR`/`TEMP`/`TMP` env vars per platform.

**State file I/O:**
- `read_state_file() -> dict[str, Any]`: Returns `json.loads(state_dir / "state.json")` or `{}` if file missing. Does NOT cache — always reads from disk so gw1+ sees gw0's incremental updates.
- `write_state_file(key: str, value: Any) -> None`: Loads current state (via `read_state_file`), updates the key, writes back via `json.dump`. Uses a module-level `_state_lock = threading.Lock()` to serialize writes within the same process (defense against concurrent fixture resolution in the same worker). Write-then-rename pattern NOT needed — `json.dump` is atomic enough for single-writer (gw0 only) scenario.
- `wait_for_state_key(key: str, timeout: float = 120.0, poll_interval: float = 0.5) -> Any`: Polls `read_state_file()` until `key` or `"error"` key appears. If `"error"` key found first, calls `pytest.skip(f"gw0 infrastructure startup failed: {state['error']}")`. If timeout expires, calls `pytest.skip(f"Timed out after {timeout}s waiting for gw0 to create '{key}'")`. Lazy imports `pytest` inside the function (only called when xdist is active).

**FileLock (lazy import of filelock):**
- `acquire_infra_lock() -> bool`: Creates a `FileLock(lock_path, timeout=0)` where `lock_path = get_state_dir() / "infra.lock"`. Calls `lock.acquire(timeout=0)` — non-blocking try. Returns `True` if lock acquired (we're gw0), `False` if `Timeout` caught (we're gw1+). `filelock` is imported lazily inside this function.
- `release_infra_lock() -> None`: Calls `lock.release()` wrapped in `contextlib.suppress(Exception)` for safe cleanup. Calls `_remove_lock_file()` if exists.

Follow ALL conventions:
- `from __future__ import annotations` as first line
- `TYPE_CHECKING` guard for `pytest` import (used in `wait_for_state_key` type hints)
- Module-level `_session_uuid: str | None = None` and `_state_lock = threading.Lock()`
- No docstrings — function signatures are self-documenting
- `os`, `json`, `threading`, `tempfile`, `contextlib`, `uuid` are stdlib imports at top
- `filelock` imported lazily in `acquire_infra_lock()`

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Environment → samstack | `PYTEST_XDIST_WORKER` env var controls worker role detection. pytest-xdist sets this; malicious user processes could set it too. |
| File system → samstack | State file (`/tmp/samstack-{uuid}/state.json`) and lock file are on shared filesystem. Other same-host processes could read/tamper if permissions allow. |
| Worker gw0 → Worker gw1+ | gw0 writes infrastructure state that gw1+ trusts. If gw0 is compromised or crashes mid-write, gw1+ could read partial/corrupt state. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-01 | Spoofing | `get_worker_id()` — env var detection | accept | `PYTEST_XDIST_WORKER` is set by pytest-xdist, not user-controlled in normal operation. A malicious user with filesystem access could set it, but at that point they already have code execution. No authentication boundary here — testing tool, not production service. |
| T-08-02 | Tampering | `write_state_file()` / `read_state_file()` — state file on shared filesystem | accept | State file in `/tmp` with default umask permissions. Another user on multi-tenant host could read endpoint URLs (LocalStack internal IPs — not production secrets). For same-host concurrent pytest sessions, UUID-based directory isolation prevents cross-session interference. |
| T-08-03 | Information Disclosure | `read_state_file()` — reads endpoint URLs from disk | accept | State file contains only internal Docker network URLs (e.g., `http://127.0.0.1:4566`, `http://sam-lambda:3001`). These are local-only, not externally routable. No secrets (access keys are hardcoded "test"/"test"). |
| T-08-04 | Denial of Service | `acquire_infra_lock()` — FileLock stale lock | mitigate | `release_infra_lock()` is called in fixture teardown (finally block). If gw0 crashes before release, the lock file is removed on next `acquire_infra_lock()` call — FileLock handles stale PID detection on Unix via fcntl (lock released when process dies). On Windows, stale locks require manual cleanup — document in README for Phase 12. Timeout on `wait_for_state_key()` (default 120s) prevents gw1+ from hanging indefinitely. |
| T-08-05 | Elevation of Privilege | `get_state_dir()` — temp directory creation | accept | Uses `tempfile.gettempdir()` which respects platform temp dirs. Directory created with `parents=True, exist_ok=True` and default permissions. No privilege escalation vector — samstack runs with the user's own permissions. |
| T-08-06 | Repudiation | State file modifications — no audit trail | accept | Testing tool, not production system. No audit requirements. State file is ephemeral (temp dir, cleaned on reboot). |
</threat_model>
